# Moodot Design System

현재 구현된 홈 화면을 기준으로 추출한 공통 디자인 규칙입니다.
보이는 사실만 정리했으며, 새로운 UI 제안이나 스타일 변경 내용은 포함하지 않습니다.

## 1. 디자인 철학

- 전체 분위기: 밝고 저채도의 파스텔 톤, 충분한 여백, 차분하고 방해 없는 인상
- UX 의도:
  빠르게 감정을 선택하고 바로 기록 행동으로 이어지도록 구성
- UX 의도:
  한 화면에 많은 선택지를 두지 않고, 핵심 행동과 최근 정보만 보여 줌
- UX 의도:
  장식 요소는 약한 블러와 얕은 그라디언트 수준으로 제한해 인지 부담을 낮춤

## 2. 레이아웃 시스템

- 상단 바 구조:
  고정형 높이 `64px`
- 상단 바 구조:
  좌측 아바타, 중앙 로고, 우측 알림 버튼의 3분 배치
- 상단 바 구조:
  내부 폭은 `max-width: 375px`, 좌우 여백은 `20px`
- 하단 네비게이션 구조:
  고정형 4탭 구조
- 하단 네비게이션 구조:
  내부 폭은 `max-width: 375px`, 내부 여백은 `16px 16px 12px`
- 하단 네비게이션 구조:
  탭은 균등 분배되며, 활성 탭만 배경 틴트가 들어감
- 전체 콘텐츠 여백 규칙:
  메인 컨테이너는 `max-width: 375px`, 좌우 `20px`
- 전체 콘텐츠 여백 규칙:
  상단 바 아래 시작을 위해 `padding-top: 80px`
- 전체 콘텐츠 여백 규칙:
  하단 네비게이션과 겹치지 않도록 `padding-bottom: 128px`
- 전체 콘텐츠 여백 규칙:
  첫 제목 블록은 `pt-8`, 감정 선택은 `pt-6`, 이후 주요 섹션은 `pt-8` 리듬 유지

## 3. 타이포그래피

- 글자 크기 계층 구조:
  메인 질문 `24px`
- 글자 크기 계층 구조:
  섹션 제목 / 주요 CTA `16px`
- 글자 크기 계층 구조:
  카드 제목 / 본문 `14px`
- 글자 크기 계층 구조:
  보조 액션 / 라벨 `12px`
- 글자 크기 계층 구조:
  메타 라벨 / 탭 라벨 `10px`
- 폰트 두께 사용 기준:
  `bold`는 화면의 1차 질문에 사용
- 폰트 두께 사용 기준:
  `semibold`는 섹션 제목, CTA, 강조 라벨에 사용
- 폰트 두께 사용 기준:
  `medium`은 보조 버튼, 탭, 메타 텍스트에 사용
- 폰트 사용 기준:
  제목/브랜드는 `Plus Jakarta Sans`
- 폰트 사용 기준:
  본문/보조 텍스트는 `Manrope`
- 텍스트 색상 구분:
  primary는 `#485058`
- 텍스트 색상 구분:
  secondary emphasis는 `#7CC4D8`
- 텍스트 색상 구분:
  muted는 `#A8B0B8`

## 4. 컬러 시스템

- 기본 배경 색상:
  `#F8FBFC`
- 카드 기본 색상:
  `#FFFFFF`
- 비활성 / 구분용 배경:
  `#F0F4F6`
- 브랜드 강조 색상:
  `#7CC4D8`, `#B8B0E0`
- 감정 색상:
  Good `#FFE8B8`
- 감정 색상:
  Bad `#F8C8C8`
- 감정 색상:
  Sad `#B0E4F8`
- 감정 색상:
  Calm `#C0ECD8`
- 색상 사용 규칙:
  감정 색상은 선택 상태, 작은 배지, 점 표시, 아이콘 배경, 약한 그라디언트에만 사용
- 색상 사용 규칙:
  전체 화면 배경은 기본 배경색을 유지하고, 강한 색 채움은 사용하지 않음
- 색상 사용 규칙:
  강조는 저채도 틴트와 낮은 불투명도로 처리

## 5. 컴포넌트 규칙

- 카드:
  기본 형태는 `rounded-xl`
- 카드:
  내부 여백은 `16px` 또는 `20px`
- 카드:
  주요 카드는 아주 약한 그림자 사용
- 카드:
  보조 카드는 그림자 대신 연한 보더 사용
- 버튼 스타일:
  1차 CTA는 전체 폭, 높이 `56px`, `rounded-full`
- 버튼 스타일:
  1차 CTA는 좌에서 우로 흐르는 브랜드 그라디언트 사용
- 버튼 스타일:
  보조 액션은 ghost 또는 텍스트 링크 스타일
- 버튼 스타일:
  아이콘 버튼은 `40x40` 원형
- 리스트 / 그리드 구조:
  홈 화면 기본 구조는 단일 컬럼 스택
- 리스트 / 그리드 구조:
  감정 선택은 4개 균등 배치의 가로 그룹
- 리스트 / 그리드 구조:
  최근 기록은 세로 카드 리스트, 카드 간 간격 `12px`
- 빈 상태:
  홈 화면에는 별도 빈 상태 UI가 보이지 않음

## 6. 인터랙션 규칙

- 클릭/터치 피드백 방식:
  `200ms` 전후의 짧은 전환 사용
- 클릭/터치 피드백 방식:
  배경색, 텍스트색, 아이콘색, 그림자, 투명도 변화 중심
- 클릭/터치 피드백 방식:
  포커스 상태는 `ring`으로 표시
- 애니메이션 스타일:
  과한 이동, 바운스, 스케일 애니메이션 없음
- 애니메이션 스타일:
  활성 상태도 강한 채움보다 옅은 틴트와 미세한 강조로 표현

## 7. Do / Don’t 규칙

- Do:
  모바일 단일 컬럼 `375px` 기준 폭을 유지한다
- Do:
  상단 고정 바와 하단 고정 네비게이션 구조를 유지한다
- Do:
  제목, 본문, 메타 텍스트의 명도 차이를 분명하게 유지한다
- Do:
  감정 색상은 작은 강조 요소에만 제한적으로 사용한다
- Do:
  카드와 선택형 요소에 둥근 모서리와 약한 깊이감을 유지한다
- Don’t:
  강한 원색이나 감정 색상을 전체 배경으로 사용하지 않는다
- Don’t:
  무거운 그림자, 강한 보더, 날카로운 모서리를 추가하지 않는다
- Don’t:
  한 화면에 경쟁하는 CTA를 여러 개 배치하지 않는다
- Don’t:
  활성 상태를 진한 단색 블록으로 처리하지 않는다
- Don’t:
  홈 화면의 정보 밀도를 높여 빠른 입력 흐름을 방해하지 않는다
